Core classes¶

Computations and transformations are based on two classes, which provide the user-side interface of computation objects.

class tigger.core.Computation(ctx, debug=False)¶

Creates a computation class and performs basic initialization for the Context object ctx. Note that the computation is unusable until prepare() or prepare_for() is called. If debug is True, a couple of additional checks will be performed in runtime during preparation and calls to computation.

The following methods are for overriding by computations inheriting Computation class.

_set_argnames(outputs, inputs, scalars)¶

Special method to use by computations with variable number of arguments. Should be called before any connections and preparations are made.

_get_operation_recorder()¶

Returns an instance of OperationRecorder class which is used in _construct_operations().

_get_argnames()¶

Must return a tuple (outputs, inputs, scalars), where each of outputs, inputs, scalars is a tuple of argument names used by this computation. If this method is not overridden, set_argnames() will have to be called right after creating the computation object.

_get_argvalues(argnames, basis)¶

Must return a dictionary with ArrayValue and ScalarValue objects assigned to the argument names.

_get_basis_for(*args, **kwds)¶

Must return a dictionary with basis values for the computation working with args, given optional parameters kwds. If names of positional and keyword arguments are known in advance, it is better to use them explicitly in the signature.

_construct_operations(basis, device_params)¶

Must fill and return the OperationRecorder object with actions required to execute the computation. See the OperationRecorder class reference for the list of available actions.

The rest is public methods.

connect(tr, array_arg, new_array_args, new_scalar_args=None)¶

Connects a Transformation instance to the computation. After the successful connection the computation resets to teh unprepared state.

Parameters:
  • array_arg – name of the leaf computation parameter to connect to.
  • new_array_args – list of the names for the new leaf array parameters.
  • new_scalar_args – list of the names for the new leaf scalar parameters.
get_nested_computation(cls)¶

Calls cls constructor with the same arguments and keywords as were given to its own constructor.

prepare_for(*args, **kwds)¶

Prepare the computation so that it could run with args supplied to __call__().

signature_str()¶

Returns a string with the signature of the computation, containing argument names, types and shapes (in case of arrays).

This is primarily a debug method.

class tigger.core.Transformation(inputs=1, outputs=1, scalars=0, derive_o_from_is=None, derive_i_from_os=None, code='${o1.store}(${i1.load});')¶

Defines an elementwise transformation.

Parameters:
  • inputs – number of input array values.
  • outputs – number of output array values.
  • parameters – number of scalar parameters for the transformation.
  • derive_o_from_is – a function taking inputs + scalars dtype parameters and returning the output dtype. If None, result_type() is used. Called when the transformation is connected to the input argument.
  • derive_i_from_os – a function taking outputs + scalars dtype parameters and returning the input dtype. If None, result_type() is used. Called when the transformation is connected to the output argument.
  • code – template source with the transformation code. See Writing a transformation section for details.
class tigger.core.operation.OperationRecorder(ctx, tr_tree, basis, base_values)¶
add_allocation(shape, dtype)¶

Adds an allocation to the list of actions. Returns the string which can be used later in the list of argument names for kernels.

add_computation(cls, *argnames, **kwds)¶

Adds a nested computation call. The computation value must be a computation with necessary basis set and transformations connected. argnames list specifies which positional arguments will be passed to this kernel.

add_kernel(template, defname, argnames, global_size, local_size=None, render_kwds=None, inplace=None)¶

Adds kernel execution to the list of actions. See Writing a computation for details on how to write kernels.

Parameters:
  • template – Mako template for the kernel.
  • defname – name of the definition inside the template.
  • argnames – names of the arguments the kernel takes. These must either belong to the list of external argument names, or be allocated by add_allocation() earlier.
  • global_size – global size to use for the call.
  • local_size – local size to use for the call. If None, the local size will be picked automatically.
  • render_kwds – dictionary with additional values used to render the template.
  • inplace – list of pairs (output, input) specifying which memory can be safely overwritten inside kernel (and therefore can point to the same physical location).
class tigger.core.ArrayValue(shape, dtype)¶

Stub object for array arguments.

shape¶

Tuple specifying the shape of the array.

dtype¶

numpy.dtype object specifying the data type of the array.

class tigger.core.ScalarValue(dtype)¶

Stub object for scalar arguments.

dtype¶

numpy.dtype object specifying the data type of the scalar.
